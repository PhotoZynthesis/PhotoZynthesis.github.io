- 这是一个副产品

这个页面其实是一个副产品。

原本我的目的是找到一套东西来方便地存放我的学习笔记，之后找了各种静态页面模板，觉得相当麻烦，最终找到了 Hexo 博客框架。Hexo 确实是简单又强大，随便配置两下就可以生成静态博客页面，配合 Github Pages 就让我这种“平民”也拥有了个像模像样的“博客”。

Hexo 很强大，可是目前来说还没有完全解决我的问题。Hexo 的目标应该是“博客”，贡献者的主题大多是那种日记形式的页面。有的主题提供了炫酷的效果及个人介绍，有的提供了实用的目录显示功能，可是目前我还没有找到完全适合于存放我的笔记的主题。

我希望有这样一个东西来存放我的笔记，它至少应具有如下特性：

  • 支持本地静态预览
  • 在笔记阅读页面提供文章列表面板，以便于快速地在不同笔记之间切换
  • 笔记阅读页面需要能根据 markdown 的标题来生成文章目录，同时不会自动提供索引，因为一直以来我都是手动写上索引

也许我可以通过自制主题来解决问题，可是目前我完全没有时间来搞这些东西，只能寄期望于日后了。

- 展望

我曾经有幸了解到大佬对博客的描述。

之所以叫博客，是因为让人看了能有所得。

目前来说我完全是一个什么都不懂的菜鸟，写不出什么有东西的文章。如果硬要写，估计这里就会成为一个“日记本”，或者根据以上对博客的描述，怕是“垃圾场”更合适。

认识的 IT 领域的朋友也很少，这更是让我看不到发布太多文章的意义。

当然眼前的东西不等于未来的全部，有机会还是希望能成为社区的一员。就当下来看，不论是这个页面还是我个人，都有很多东西需要改观。
